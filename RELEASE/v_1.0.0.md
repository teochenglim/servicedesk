# v1.0.0 — ServiceDesk core: UI, multi-tenant, multi-user login

Theme: a complete, runnable ITSM ticketing system — single Go binary + static HTML/HTMX UI, with real multi-tenant customer support and role-based access, on any of SQLite/MySQL/PostgreSQL.

## Shipped

**Core ticketing**
- Ticket CRUD with the explicit state machine (New/In Progress/Resolved/Closed/Rejected, forced Reject→Closed cascade, restricted Reopen-from-Closed).
- Queues with hierarchy (`ParentID`) and per-queue default priority/category.
- Queue membership: pickup (self-assign) requires membership; assign/transfer to another engineer is QueueAdmin+ only and bypasses membership (a deliberate "transfer" semantic vs. "pickup").
- Internal/external notes with Markdown (goldmark + GFM) and client-side syntax highlighting.
- Incident + RCA labels, Problem Management records linking multiple tickets.
- Watch/unwatch, including as the multi-tenant sharing mechanism (see below).
- Full-text search across tickets + notes, implemented per database dialect.
- Dynamic custom fields (definitions only wired through the admin UI so far; not yet rendered into the ticket-create form).

**Roles & multi-tenancy**
- Roles: Customer, Engineer, QueueAdmin, SystemAdmin (Tier1/2/3 collapsed into a single Engineer role — support tier is a queue-membership property, not a global rank).
- Organizations (`Organization`/`OrgMembership`), self-referencing `ParentID` for a future Group→Company→Department hierarchy.
- Org-scoped Customer login (org name + username + password); internal staff are unscoped.
- Customer ticket visibility: own tickets + same-org tickets shared via watch. Enforced at the query layer, not just the UI.
- Admin UI for organizations (`/admin/orgs`) and queues (`/queues`), including membership management for both.

**Automation**
- Lightweight workflow engine: `condition`, `user_input`, `http_request`, `template_render`, `add_note`, `auto_assign`, `webhook`, `approval`, `notify` steps, one interpreter for both ad-hoc rule workflows and the Runbook Hook.
- Runbook Hook: pause for user input mid-ticket, call an external API, render a note from a template — see [DESIGN/04_design_runbook_hook.md](../DESIGN/04_design_runbook_hook.md).
- Approvals: named-role approval steps that can reject and stop a workflow.
- Webhooks: signed (HMAC-SHA256) outbound delivery with a durable outbox and exponential-backoff retry (max 3 attempts).
- SSE real-time updates to ticket watchers/assignees.

**Platform**
- Multi-database: SQLite (default), MySQL, PostgreSQL — all three validated live via their `docker-compose-*.yml`, not just SQLite.
- YAML config file support (`-config` / `SERVICEDESK_CONFIG_FILE`) alongside env vars, env always wins.
- Structured logging with 5 levels (`fatal`/`error`/`warning`/`info`/`debug`).
- Prometheus metrics at `/metrics`; `/health` readiness endpoint.
- Docker (multi-stage, distroless, CGO-free static binary), 3 docker-compose variants, Kubernetes manifests (ConfigMap/Secret/StatefulSet-Postgres/Deployment/Service/Ingress/HPA), GitHub Actions CI (build/vet/test + push to GHCR on main/tags).
- 26 automated tests: state-machine unit tests + full-stack integration tests via a real HTTP server against in-memory SQLite.

## Explicitly deferred (not in this version)

- LDAP/OAuth2 (env var parsed, not wired to a real directory).
- Dynamic custom fields are not yet rendered on the ticket-create/edit form (definitions exist, admin CRUD exists).
- SLA due-date automation (the field exists on `Ticket`; nothing sets or enforces it yet).
- Organization hierarchy (`ParentID`) is schema-ready but no UI/logic treats a parent org specially yet (e.g. no "see all child-org tickets" rollup for a Group-level user).

## Bugs found and fixed during this build

- **Approval rejection didn't stop the workflow**: `Engine.Resume` unconditionally incremented `StepIndex`, skipping the `approval` step's own rejection check on re-entry. Fixed by leaving `StepIndex` untouched in `Resume` and letting each step type's own "have I been resumed?" check in `runStep` decide whether to advance. Covered by `TestApproval_RejectionStopsWorkflow`.
- **`next_run_at` comparisons were always false**: binding a Go `time.Time` to sqlite produced RFC3339 (`T` separator), which never compares correctly against SQL `CURRENT_TIMESTAMP`'s space-separated format — workflow tasks and webhook deliveries would enqueue but never get claimed. Fixed by storing `next_run_at` as a Unix-epoch `int64` and comparing against `time.Now().Unix()` explicitly.
- **MySQL: "BLOB/TEXT column used in key specification without a key length"**: GORM defaults an untagged Go `string` field to `TEXT` on MySQL, which can't be indexed without a prefix length. Fixed by adding explicit `size:` tags to every `uniqueIndex`-tagged string field.
- **PostgreSQL 18 container reported unhealthy**: the postgres 18+ images changed the expected data-volume mount point from `/var/lib/postgresql/data` to `/var/lib/postgresql`. Fixed the compose file's volume mount.
- **SQLite in Docker: "unable to open database file"**: the distroless `nonroot` user couldn't write to a fresh named volume mounted at `/data` (Docker initializes an empty named volume owned as created, not matching the image's intended owner). Fixed by pre-creating `/data` in the build stage and `COPY --chown=65532:65532`-ing it into the runtime image, so the volume inherits correct ownership on first creation.
