# v1.0.10 — Persona-Driven UX Redesign: RBAC foundation, stage tracking, attachments & rich text

Theme: the first slice of the persona-driven redesign in [DESIGN/08_design_ux.md](../DESIGN/08_design_ux.md) — the shared plumbing every per-role home screen in [v_1.0.11.md](v_1.0.11.md) depends on, plus the two cross-cutting shared components (attachments, rich text) that were low-risk enough to ship standalone.

## Shipped

- **RBAC foundation** ([DESIGN/02](../DESIGN/02_design_roles_and_tenancy.md) §2.1.1): `RoleQueueAdmin` renamed to `RoleManager`; queue ownership (`CapQueueOps`) is now a capability check (`Role.Can`), not inherited via the linear role rank — SystemAdmin no longer natively holds it (reaches it only through the Sudo-as mechanism planned in v1.0.11).
- **Stage-tracking overlay + shared Ticket Progress Bar** ([DESIGN/03](../DESIGN/03_design_functional_requirements.md) §3.1.2b, [DESIGN/08](../DESIGN/08_design_ux.md) §8.2): `DetectedAt/AckedAt/MitigatedAt/ResolvedAt/ReopenCount` added to `Ticket` as a purely additive overlay — the existing state machine (`internal/service/statemachine.go`) is untouched. One-time backfill from `event_logs` for pre-existing tickets. New `POST /tickets/{id}/mitigate` action. New `servicedesk_ticket_mtt{d,a,m,r}_seconds` Prometheus histograms.
- **Agent actor identity**: new `RoleAgent` (shares Engineer's capability rank), long-lived API-token auth (`sdat_<id>.<secret>`) as a narrow, swappable auth path alongside the human JWT+cookie flow — designed so a future OIDC/external-IdP login for Agent replaces just that branch.
- **Attachments** ([DESIGN/08](../DESIGN/08_design_ux.md) §8.7): stored as a DB blob column (`Attachment.Data`), not local disk or an object store — `StorageBackend` is a discriminator column reserved for a future RustFS/S3 backend. Visibility: staff see everything; a Customer's own upload is private to them even among other Customers who can see the same ticket (e.g. a same-org watcher); a staff-uploaded external attachment stays visible to any Customer who can see the ticket. Images render as inline thumbnails; everything else forces a download. Referencing an uploaded file's URL from Markdown (`![x](/attachments/N)`) already renders inline securely — no bundling code needed, the download endpoint re-checks visibility per request.
- **Rich text**: Toast UI Editor (vendored, WYSIWYG⇄Markdown toggle built in) wired into both ticket submission and the note composer, for every role — still stores plain Markdown, so webhooks/future AI/KB consumers are unaffected.

## Explicitly deferred to v1.0.11

Per-role home screens (Engineer/Customer/Manager/ServiceDeskAdmin), Sudo-as, the Manager dashboard, and the AI/KB shared components — see [v_1.0.11.md](v_1.0.11.md).

Not in scope: the existing Problems/RCA feature (`/problems`) is untouched.
