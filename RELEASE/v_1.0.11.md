# v1.0.11 — Persona-Driven UX Redesign: per-role home screens, Sudo-as, AI/KB

Theme: the remainder of the persona-driven redesign in [DESIGN/08_design_ux.md](../DESIGN/08_design_ux.md), built on top of the RBAC/stage-tracking/attachments/rich-text foundation shipped in [v_1.0.10.md](v_1.0.10.md). This absorbs [v_3.0.0.md](v_3.0.0.md)'s "AI/Agent collaboration" placeholder — the AI Ticket Intelligence Panel and drafting work below answer the open questions that file raised, so v_3.0.0 is superseded by this file rather than being a separate initiative.

## Already shipped (don't re-do — see v_1.0.10.md)

RBAC capability model, stage-tracking overlay + Progress Bar, `RoleAgent` + API-token auth, attachments (DB-blob storage, Customer-upload privacy), Toast UI Editor rich text.

## Planned / not started

- **Engineer view rebuild** ([DESIGN/08](../DESIGN/08_design_ux.md) §8.5): two tabs ("My tickets" / "Available in my queues"), stage-age-first default sort, "Raise ticket for a user," lightweight Transfer picker. Today every role still shares the one `tickets_workspace.html`.
- **Customer view rebuild** ([DESIGN/08](../DESIGN/08_design_ux.md) §8.4): full-width Submit button, plain-English progress bar (no Ack/Mitigate jargon), Close/Reopen/Add-note always visible regardless of status.
- **Service Desk Manager**: dashboard (per-queue open/breaching counts, engineer load, MTTx trend), activity list (most-recently-updated scan view), per-queue SLA targets, `Ticket.LastActivityAt` denormalized column ([DESIGN/08](../DESIGN/08_design_ux.md) §8.6).
- **ServiceDeskAdmin + Sudo-as** ([DESIGN/02](../DESIGN/02_design_roles_and_tenancy.md) §2.5, [DESIGN/08](../DESIGN/08_design_ux.md) §8.3): session-scoped acting-as (JWT `SudoBy` claim), persistent banner, promoted system audit log. Highest-risk phase from a security standpoint — run `/security-review` before merging.
- **AI-assisted drafting + AI Ticket Intelligence Panel** ([DESIGN/08](../DESIGN/08_design_ux.md) §8.8–8.9): needs an LLM/framework decision first (open question carried over from v_3.0.0.md) — recommend a small `internal/llm.Client` interface seam regardless of vendor. Versioned snapshots, human-edit-outranks-regeneration semantics, Suggest Split.
- **Knowledge Base feedback loop** ([DESIGN/08](../DESIGN/08_design_ux.md) §8.10): human curation gate before anything publishes to a customer-facing surface, suggestion-matching on the structured extraction at submission and triage.

## Suggested build order

1. Engineer view rebuild (highest ticket-touch frequency).
2. Customer view rebuild (lowest risk, validates the visual language externally).
3. Manager role, dashboard, activity list, queue ownership UI — ahead of ServiceDeskAdmin since ServiceDeskAdmin's redesign is defined in contrast to it.
4. ServiceDeskAdmin view, including Sudo-as and the audit log.
5. AI draft buttons + AI Ticket Intelligence Panel.
6. Knowledge Base feedback loop (needs a real backlog of curated snapshots from step 5).

Not in scope: the existing Problems/RCA feature (`/problems`) is untouched.
