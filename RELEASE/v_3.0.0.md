# v3.0.0 — Quick-wins backlog

Theme: small, self-contained gaps left behind by earlier versions — each independently shippable, none requiring a design pass first. Consolidated here after a sweep of every `v_1.0.*.md`/`v_2.*.md` "Explicitly deferred"/"Planned / not started" section to check what's actually still open (several were already resolved by later versions — e.g. v1.0.0's SLA-automation deferral was closed out by v1.0.11/v2.0.0 — and aren't repeated here). Larger initiatives needing real design work are tracked separately in [v_3.1.0.md](v_3.1.0.md).

Not carried forward (judged not worth a backlog slot): v1.0.8's demo-mode cosmetics (scripted tour UI, sample webhook data requiring a real external endpoint) and v1.0.9's README image polish — noted here so they're not silently dropped, but they're not real product gaps.
