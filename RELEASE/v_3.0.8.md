# v3.0.8 — favicon

Theme: the site had no favicon at all (bare tab icon / browser default). Added one.

## Built

- `web/static/favicon.svg` — a small headset icon (support/service-desk symbol) on a rounded dark-teal square, using the existing brand palette from `tokens.css` (`--tw-teal-900` background, `--tw-coral-500` headset, white mic boom/dot). Generated programmatically (Python script computing the arc/rect geometry, not hand-drawn) rather than sourced from an icon set, so it's a plain inline `<svg>` with no external dependency.
- Linked via `<link rel="icon" type="image/svg+xml" href="/static/favicon.svg">` in both `web/templates/layout.html` (the shared authenticated-app head) and `web/templates/login.html` (which has its own separate `<head>` block — easy to miss since it doesn't inherit from `layout.html`).
- No embed changes needed: `web/embed.go`'s `//go:embed static` already embeds the whole `static/` tree recursively, so the new file ships in the binary automatically.

## Verification

- `make vet` clean.
- Rendered the SVG standalone via `rsvg-convert` and visually confirmed the headset icon reads correctly at both 256px and small (16-32px favicon) sizes before wiring it in.
